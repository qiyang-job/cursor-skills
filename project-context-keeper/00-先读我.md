# 项目上下文守护 skill

> 目的：把“AI 容易遗忘项目上下文”变成一套可维护的项目记忆系统。  
> 用法：把整个文件夹直接放进项目里，让 AI 先读 `SKILL.md`，再按规则维护项目的 `.ai-context/`。

---

## 1. 这是什么版本

当前是 **`v2-lite`**，相比最早版本，重点补了 4 个能力：

- **快速入口**：先读 `00-current-context.md`
- **规则层**：新增 `07-conventions.md`
- **经验层**：新增 `08-lessons-learned.md`
- **体积控制**：明确归档与长度约束，避免文档膨胀

这次升级不是为了“多几份文档”，而是为了让 AI：

- 每次更快恢复项目状态
- 更少漏读关键规则
- 更少重复犯同类错误
- 长时间使用后仍然保持可读

---

## 2. 最推荐的使用方式

### 场景 A：项目刚进入多轮开发

让 AI 先读：

- `SKILL.md`
- `references/01-上下文文件规范.md`
- `references/02-更新与压缩规则.md`

然后初始化项目里的 `.ai-context/`。

### 场景 B：隔了一段时间恢复开发

让 AI 先读项目里的：

- `.ai-context/00-current-context.md`
- `.ai-context/00-project-overview.md`
- `.ai-context/01-architecture-map.md`
- `.ai-context/02-current-state.md`
- `.ai-context/04-session-handoff.md`
- `.ai-context/07-conventions.md`

如果怀疑在重复踩旧坑，再补读：

- `.ai-context/08-lessons-learned.md`

### 场景 C：做完一轮改动准备收尾

至少更新：

- `.ai-context/00-current-context.md`
- `.ai-context/02-current-state.md`
- `.ai-context/04-session-handoff.md`

如有需要，再补：

- `.ai-context/03-decision-log.md`
- `.ai-context/05-open-questions.md`
- `.ai-context/06-change-log.md`
- `.ai-context/07-conventions.md`
- `.ai-context/08-lessons-learned.md`

## 3. 和旧版相比最大的变化

### 不再默认“全量读一遍”，而是先读一个超轻快照

核心入口改成：

- `00-current-context.md`

这份文件只回答 5 件事：

- 现在项目一句话状态是什么
- 当前主任务是什么
- 当前阻塞是什么
- 下一步最该做什么
- 还需要继续读哪些文件

### 单独分出“规则层”和“经验层”

这样可以避免：

- 状态写在状态文件里
- 规则混在交接里
- 教训散落在聊天里

---

## 4. 一个很重要的提醒

### 不要把这个 skill 当成“聊天备份工具”

正确目标是：

- 记住未来还会继续用到的项目事实
- 记住当前状态
- 记住决策原因
- 记住稳定规则
- 记住已经踩过的坑

不是：

- 把每轮对话逐字保存下来

---

## 5. 给 AI 的推荐开场句

```md
先不要直接写代码。

请先阅读这个文件夹里的 `SKILL.md`，并按其中的规则帮助我同步当前项目上下文。

如果项目里已经有 `.ai-context/`，请先读取：
1. `.ai-context/00-current-context.md`
2. `.ai-context/00-project-overview.md`
3. `.ai-context/01-architecture-map.md`
4. `.ai-context/02-current-state.md`
5. `.ai-context/04-session-handoff.md`
6. `.ai-context/07-conventions.md`

然后输出：
1. 当前项目目标
2. 当前系统主链路 / 架构概览
3. 当前主任务
4. 最近关键变更
5. 当前阻塞点 / 未决问题
6. 当前需要遵守的关键规则
7. 这轮最合理下一步

如果项目里还没有 `.ai-context/`，请先帮我初始化一套 v2-lite 项目记忆包。
```

---

## 6. 最后一句提醒

**让 AI 长期记住项目，不靠一次塞更多上下文，而靠把重要上下文持续沉淀成：短快照 + 稳定事实 + 当前状态 + 关键规则 + 经验教训。**
