# UI 质量目标与工作方式

## 1. 先定义“高质量”是什么意思

高质量 UI 不等于：

- 颜色更多
- 阴影更重
- 动画更多
- 页面更花哨

高质量 UI 更接近这些结果：

- 一眼知道重点在哪里
- 主要操作路径更顺
- 组件状态更完整
- 页面节奏更舒服
- 风格统一且有产品感
- 有细节，但不过度炫技

---

## 2. 推荐工作顺序

建议总是按这个顺序工作：

1. **先看目标用户和页面任务**
2. **先看结构和信息层级**
3. **再看视觉样式是否统一**
4. **再补交互反馈和状态**
5. **最后补动效、细节和性能**

顺序反了，通常就会变成“表面精致，实际难用”。

---

## 3. 本 skill 重点提升的 6 个维度

### 视觉层级
- 标题、正文、辅助信息、操作区要能快速区分

### 页面节奏
- 模块之间要有呼吸感和组织感

### 组件完成度
- 不只默认态，还要考虑各种反馈状态

### 交互可感知性
- 用户每个操作后都应该得到清晰反馈

### 动效自然度
- 动效要像“状态变化的延续”，而不是“表演”

### 整体统一性
- 圆角、边框、阴影、色彩、文案语气要来自同一语言系统

---

## 4. 一个实用判断标准

### 如果去掉颜色和插画，这个页面仍然有清晰层级和顺滑操作，它的质量通常才是真的上来了。
