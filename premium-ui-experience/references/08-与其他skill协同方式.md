# 与其他 skill 协同方式

## 1. 和 `01-开发约束skill` 的关系

推荐顺序：

1. 先用 `01-开发约束skill`
2. 再用 `03-高质量UI体验skill`

原因：

- 先把范围收住，避免界面在错误目标上越做越复杂
- 先明确最小闭环，再决定哪些地方值得重点打磨

---

## 2. 和 `项目上下文守护skill` 的关系

推荐关系：

- `03-高质量UI体验skill` 负责把 UI / UX 做好
- `项目上下文守护skill` 负责记住项目当前状态、规则和最近变更

建议把以下内容沉淀到 `.ai-context/`：

- 重要 UI 决策 -> `03-decision-log.md`
- 项目稳定设计规则 -> `07-conventions.md`
- 容易重复犯的 UI 问题 -> `08-lessons-learned.md`

---

## 3. 和项目级 `design.md` 的关系

两者分工应当明确：

- **本 skill**：方法、流程、检查和打磨方式
- **`design.md`**：这个项目自己的审美、气质、参考、偏好和禁忌

不要让 `design.md` 变成这个 skill 的重复副本。

---

## 4. 和设计类技能的叠加建议

如果环境支持额外设计技能，可按问题类型叠加：

- 需要首版高质量页面：`frontend-design`
- 需要改善布局：`arrange`
- 需要改善排版：`typeset`
- 需要改善色彩：`colorize`
- 需要改善动效：`animate`
- 需要做收尾：`polish`
- 需要补边界和鲁棒性：`harden`
- 需要兼顾性能：`optimize`
- 需要系统检查：`audit`

---

## 5. 一个推荐组合

如果你想让一个项目同时更稳、更美、更不容易忘：

1. `01-开发约束skill/`
2. `03-高质量UI体验skill/`
3. `项目上下文守护skill/`

一句话理解：

- 第 1 个负责 **别一开始就做歪**
- 第 3 个负责 **把界面和体验真正做细**
- 第 2 个负责 **别做着做着又忘了**
