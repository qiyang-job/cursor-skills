# 组件状态与细节打磨

## 1. 不要只做默认态

高质量组件至少要考虑：

- 默认态
- hover
- focus
- active / pressed
- disabled
- loading
- empty
- error
- success

如果只做默认态，界面很容易停留在“演示稿质量”。

---

## 2. 细节一致性很重要

重点检查：

- 按钮高度是否统一
- 输入框边界是否统一
- 圆角和阴影是否来自同一语言
- 图标尺寸和线条风格是否一致
- 标签、辅助文案、提示信息是否有统一层级

---

## 3. 状态反馈要有明确差异

例如：

- disabled 不只是变灰，还要看起来不可交互
- loading 不只是转圈，还要让用户知道当前在等什么
- error 不只是变红，还要给出明确提示
- success 不只是闪一下，还要让用户知道操作真的完成了

---

## 4. 注意边界问题

高质量界面往往赢在这些地方：

- 长文本不炸布局
- 空数据不显得像坏掉
- 错误提示不抢主线
- 加载时不跳动
- 小屏下仍有秩序

---
